# Dungeon Steward Character / Faction Prompt Pack

Use this pack for the approved character/faction-first direction defined in `design/art-bible.md`.

This is a concept-generation pack, not a final-runtime-asset pack. The models are useful servants, but not art directors.

## North Star

Generate images that read as:
- deadpan fantasy absurdism
- courtly glamour versus frontier grime
- browser-first game art
- readable silhouette
- coherent social world
- strategy-first presentation

Do **not** generate for:
- parody comedy
- meme culture
- pinup-first fantasy
- random monster collage
- final tiny UI assets

---

## Global Style Guidance

### Shared positive tags
- browser-first game art
- readable silhouette
- controlled value grouping
- portrait-friendly composition
- strong focal hierarchy
- restrained fantasy detail
- elegant absurdity
- deadpan fantasy absurdism
- coherent faction design
- high clarity concept art
- dark neutral or parchment-friendly background

### Shared negative tags
- text
- watermark
- signature
- border
- frame
- muddy composition
- unreadable silhouette
- low contrast focal point
- over-rendered background
- random extra limbs
- costume clutter
- parody comedy
- meme expression
- pinup pose
- fetish styling
- oversaturated neon

### Runtime truth
Do not ask the model for final production pixel art and expect it to behave like a civilized grid.

Preferred process:
1. generate concept base
2. select silhouette, materials, and composition
3. reduce complexity manually
4. rebuild or finish for runtime use by hand

## Prompt Construction Rules

### Good prompt behavior
Favor:
- one subject
- one role
- one material family
- one mood contrast
- one strong composition request

### Bad prompt behavior
Avoid:
- listing every faction at once unless explicitly doing a lineup
- adding too many materials
- mixing every color family in one image
- demanding pixel art + photoreal rendering + ornate poster + transparent sprite all at once

The model is suggestible, not wise.

---

## Workflow Recommendations

### HUD portrait rule (hard)
- For combat HUD portrait targets, prompts must explicitly request:
  - bust or head-and-shoulders composition
  - face clearly readable at small size
  - strong silhouette in a tight crop
  - dark or simple background for easy extraction
- Do NOT accept:
  - wide scene illustrations for portrait slots
  - full-body sprite framing when the integration target is a portrait panel
  - compositions where props/environment dominate more than the face and torso

### Prototype button icon rule (hard)
- Do not use general image generation to produce prototype HUD button icons unless the icon meaning is semantically exact at tiny size.
- `Pass` must clearly read as skip/end-turn/forward-turn.
- `Restart` must clearly read as refresh/reset/retry.
- Reject outputs that read as:
  - death / urn / memorial
  - home / house / up-arrow / navigation
  - generic decorative heraldry without a clear interaction meaning
- If semantic clarity is weak, use text-only buttons and wait for hand-authored UI glyphs.

### Use FLUX Schnell for:
- characters
- faction sheets
- enemy family sheets
- boss ideation
- hub moods

### Use SDXL relic workflow for:
- relics
- charm objects
- investment objects
- UI motif ideation

## Approval Checklist For Curated Selects

Promote an image only if:
- it has a clear focal point
- it survives a card thumbnail test
- it does not look like parody or fetish bait
- it fits the art bible's material and color language
- it strengthens faction identity without destroying gameplay clarity

If it merely looks expensive, reject it. The model can produce costly-looking mistakes all day.

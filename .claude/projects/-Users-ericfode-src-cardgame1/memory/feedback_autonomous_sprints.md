name: Autonomous Sprint Work
description: User prefers continuous autonomous execution during sprint work, checking in only at the end of the sprint rather than asking at each task boundary.
type: feedback
---

When working through sprint tasks, continue autonomously without asking for permission at each step. Check in at the end of the sprint with a summary.

**Why:** User explicitly said "continue without asking. checkin at the end of the sprint." — they trust the approved plan and don't want friction between tasks.

**How to apply:** After the sprint plan is approved and work begins, execute all tasks end-to-end. Only pause if genuinely blocked or if a decision would contradict an existing design doc. Present a full sprint summary at the end.
